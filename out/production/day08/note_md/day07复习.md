### 方法
*   方法：描述对象的行为特征或功能特征
    *   封装的功能代码块，方便重复利用
*   方法的声明：
    * 修饰符 返回值类型 方法名（参数列表：数据类型 变量名，数据类型 变量名2，...）
    ``` java
        {     
            代码块
              return;//返回的数据
        }
    
    ```   
* 方法的分类：
    * 类方法(静态方法)：有static修饰的方法
    * 实例方法：没有static修饰符修饰的方法
    
* 方法的调用：
    * 方法不调用不执行，调用一次执行依次。
    * 类方法 ：类名.方法名()
        * 或实例对象.方法名() 不推荐
    * 实例方法：
        * 只能是对象.方法名()
    * 同一个类中，方法和成员变量的调用，可以省略 对象名和类名 直接调用
    ```java
    class Demo{
      String name;
      static int age;
      public void show(){
          System.out.println(name);
          System.out.println(age);
     }
  }
    ```
* 方法调用的内存分析：
    * 方法被调用，会入栈，在栈中为每个方法开辟独立空间，方法调用完毕，立刻释放空间
    > 见day07images
    
* 方法参数的值传递机制：
    * 当形参是基本数据类型时，传递的是数据值，修改参数后，实参不会被修改
    * 当形参是引用数据类型时，传递的时地址值，修改参数后，实参会被修改(此地址上的数据)   
    
* 方法重载：overload
    * 在同一个类中，方法名相同，参数列表不同，称为方法重载
    * 参数列表不同指的是：
        * 参数个数不同
        * 参数类型不同
        * 参数类型顺序不同
        
* 可变参数：
    * 方法中的参数个数可以任意个，类型相同。  
    > public void test(int...num){}
    * 此时num就是可变参数，本质上是一个数组
